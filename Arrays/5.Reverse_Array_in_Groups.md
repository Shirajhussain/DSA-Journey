# GFG - Reverse Array in Groups

**Platform:** GeeksforGeeks
**Difficulty:** Medium
**Topic:** Arrays, Two Pointers

---

# Problem Statement

Given an integer array `arr[]` and an integer `k`, reverse every consecutive group of `k` elements.

If the number of remaining elements is less than `k`, reverse all of them.

---

# Key Observation

The array can be divided into multiple groups of size `k`.

Each group is independent, so we simply reverse one group at a time.

For the last group, if fewer than `k` elements remain, reverse the entire remaining part.

---

# Approach

Traverse the array with a step size of `k`.

For each group:

1. Determine the starting index.
2. Determine the ending index.
3. Reverse that portion using two pointers.

---

# Example

### Input

```text
arr = [1,2,3,4,5]
k = 3
```

### Group 1

```text
[1,2,3] [4,5]
```

Reverse

```text
[3,2,1] [4,5]
```

---

### Group 2

```text
[3,2,1] [4,5]
```

Reverse remaining elements

```text
[3,2,1] [5,4]
```

Output

```text
[3,2,1,5,4]
```

---

# Algorithm

1. Traverse the array in steps of `k`.
2. Set:

   * `start = i`
   * `end = min(i + k, n)`
3. Reverse elements from `start` to `end - 1`.
4. Repeat until all groups are processed.

---

# Time Complexity

Each element is visited only once.

**O(n)**

---

# Space Complexity

No extra array is used.

**O(1)**

---

# Key Interview Points

* Process the array one group at a time.
* Reverse each group using two pointers.
* The final group may contain fewer than `k` elements.
* The algorithm performs the reversal in-place.
* No extra space is required.
