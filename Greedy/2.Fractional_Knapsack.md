# GFG - Fractional Knapsack

**Platform:** GeeksforGeeks
**Difficulty:** Medium
**Topic:** Greedy, Sorting

---

# Problem Statement

Given two arrays:

* `val[]` representing the value of each item.
* `wt[]` representing the weight of each item.

Also given the capacity of a knapsack.

Find the maximum total value that can be obtained by putting items into the knapsack. You are allowed to take a fraction of an item.

---

# Approach

The Greedy strategy is to always pick the item with the highest **value per unit weight** first.

## Idea

1. Calculate the value-to-weight ratio for every item.
2. Sort the items in decreasing order of this ratio.
3. Traverse the sorted items:

   * If the entire item fits, take it.
   * Otherwise, take only the fraction that fits and stop.

This strategy guarantees the maximum possible value.

---

# Algorithm

1. Store each item as `(value, weight)`.
2. Sort the items based on:

   ```
   value / weight
   ```

   in descending order.
3. Initialize `sum = 0`.
4. Traverse all items:

   * If the weight is less than or equal to the remaining capacity:

     * Take the complete item.
   * Otherwise:

     * Take the required fraction.
     * Stop.

---

# Time Complexity

* Sorting: **O(n log n)**
* Traversal: **O(n)**

Overall:

**O(n log n)**

---

# Space Complexity

**O(n)**

Extra space is used to store the `(value, weight)` pairs.

---

# Key Interview Points

* Fractional Knapsack uses a **Greedy** approach.
* Sort items by **value/weight ratio**.
* Take the highest ratio items first.
* Unlike 0/1 Knapsack, fractional items are allowed.
* Because fractions are allowed, the Greedy strategy produces the optimal solution.
