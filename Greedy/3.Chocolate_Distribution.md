# GFG - Chocolate Distribution Problem

**Platform:** GeeksforGeeks
**Difficulty:** Easy
**Topic:** Greedy, Sorting, Sliding Window

---

# Problem Statement

Given an array `arr[]` where each element represents the number of chocolates in a packet and an integer `m` representing the number of students.

Distribute exactly one packet to each student such that the difference between the maximum and minimum chocolates received is minimized.

Return the minimum possible difference.

---

# Approach

The main observation is:

* After sorting the packets, the best choice will always be **m consecutive packets**.
* We only need to check every window of size `m`.

### Why?

Suppose the sorted array is:

```
1 3 4 7 9 9 12 56
```

If we choose `m = 5`, the possible windows are:

```
[1 3 4 7 9]
[3 4 7 9 9]
[4 7 9 9 12]
[7 9 9 12 56]
```

The difference for each window is simply:

```
last element - first element
```

The minimum of these differences is the answer.

---

# Algorithm

1. Sort the array.
2. Initialize two pointers:

   * `i = 0`
   * `j = m - 1`
3. While `j < n`:

   * Compute `arr[j] - arr[i]`.
   * Update the minimum difference.
   * Move both pointers one step ahead.
4. Return the minimum difference.

---

# Time Complexity

* Sorting: **O(n log n)**
* Sliding Window: **O(n)**

Overall:

**O(n log n)**

---

# Space Complexity

**O(1)**

(Excluding the space used internally by the sorting algorithm.)

---

# Key Interview Points

* Sort the array first.
* The optimal distribution is always a contiguous window after sorting.
* Use a sliding window of size `m`.
* Keep track of the minimum difference among all windows.
